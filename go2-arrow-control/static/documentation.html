1. System Overview

The GO2 Arrow Control System allows you to control a Unitree GO2 robot using hand-drawn arrows recognized by a computer vision model. The workflow consists of:

  1. Training: Use Google Teachable Machine to train a model to recognize arrows.
  2. Uploading: Upload the trained `.tflite` model to this system.
  3. Controlling: Log in to the control dashboard, select your model, and drive the robot.

2. Training Guide

Your model must be trained to recognize exactly these 5 classes (names must match exactly):

  • Forward - Robot moves forward
  • Right - Robot turns right (yaw)
  • Left - Robot turns left (yaw)
  • Rotate - Robot spins in place (or moves backward depending on config)
  • Idle - Robot stands still (Default state)

3. Control Parameters

These settings are available in the Control Dashboard to fine-tune performance:

Parameter Range Description
Confidence Threshold 50% - 95% Minimum certainty required for a prediction to be accepted.
Higher = Fewer accidents, harder to trigger. Lower = More responsive, prone to noise.
Max Speed 0.1 - 0.5 m/s Limits the top speed of walking and turning.
Start low (0.1) for testing. Increase only when confident.
command_interval Advanced Settings Minimum time interval (in seconds) between commands sent to the robot. Higher value = Slower reaction but smoother motion.
buffer_size Advanced Settings Size of the sliding window used for prediction smoothing. Larger buffer = More stable but more lag.
consensus_required Advanced Settings Number of matching predictions required within the buffer to trigger a move. Prevents "flickering" commands.

4. Safety Features

  • Dead Man's Switch: The robot automatically stops if no command is received for 2 seconds.
  • Inference Stop: Clicking "Stop Control" immediately halts the robot.
  • Emergency Stop: The Red "EMERGENCY STOP" button cuts all movement commands instantly.
